Tutorial

Para garantir a pureza do código principal, fornecemos apenas a mais pura função de varredura automática de código.

QScan Ele pode ser chamado diretamente usando a linha de comando ou usado como um módulo Node, que é muito flexível de usar ~

Configuração

QScan A digitalização requer uma configuração, que pode ser escrita em ~/.qscanrc No arquivo, ou no código, o formato desta configuração é o seguinte： 

{
    "wx-default": { // key O valor é o nome modal
        "udid": "HICMHMZTTW8DFI59", // ID do dispositivo Android, visualizar por meio do comando de serviço adb
        "port": "4723", // O número da porta
        "opts": {
            "user": "xxx", // Nome de usuário do WeChat
            "pass": "xxx" // Senha do WeChat
        },
        "checkApp": false // Verifique se deseja instalar o APP WeChat e sua versão
    }
}

Nota: o nome de usuário do WeChat deve usar a conta do WeChat, número de telefone celular é proibido

QScan Fornecer oficialmente Modal, especificar Modo de leitura de código，Atualmente, suporta apenas código de verificação WeChat：

wx-default (Código de verificação do WeChat)

type nome Características
ide-login-scan Login das ferramentas do desenvolvedor
backstage-login-scan Login em segundo plano do WeChat

Além disso, você também pode desenvolver seu próprio Modo de leitura de código： Modo personalizado

Método de chamada

Chamada de linha de comando

Depois de instalar o QScan globalmente, você pode usar a linha de comando para chamá-lo e agora oferece suporte a três comandos：

Commands:
  doctor [options]  Verifique o ambiente operacional
  scan [options]    Leia o código QR

1. qscan doctor Verifique o ambiente operacional

Options:
  -c, --customModel <customModel>  Caminho do modo personalizado, vazio por padrão
  -r, --runConfig <runConfig>      Execute o caminho absoluto da configuração, o padrão é ~/.qscanrc
  -h, --help                       output usage information

Configurado ~/.qscanrc Depois de executar：

qscan doctor

2. qscan scan Leia o código QR

Options:
  -c, --customModel <customModel>  Caminho do modo personalizado, vazio por padrão
  -r, --runConfig <runConfig>      Configuração de execução, o padrão é ~/.qscanrc
  -m, --model <model>              Modo de digitalização usado
  -t, --type <type>                Tipo de código de digitalização usado
  -h, --help                       output usage information

Configurado ~/.qscanrc Depois de executar：

qscan scan -m 'wx-default' -t 'ide-login-scan'

Use como um módulo de node

const QScan = require('qscan'); // Apresentar a ferramenta qscan

// Instancie um objeto de verificação
const scan = new QScan({
    // model Configuração
    modelOpts: {
        'wx-default': { // model nome
            udid: 'HICMHMZTTW8DFI59', // ID do dispositivo Android, visualizar por meio do comando de serviço adb
            port: '4723', // O número da porta
            opts: {
                user: 'xxx', // Nome de usuário do WeChat
                pass: 'xxx' // Senha do WeChat
            }
        }
    }
});

Instanciamos um objeto QScan, denominado scan, este objeto possui os seguintes métodos：

scan.run({ modelName, type }, cb) Leia o código QR:

scan.run(
    {
        modelName: 'wx-default', // model nome
        type: 'ide-login-scan' // Tipo de código de digitalização
    },
    err => {
        if (err) {
            console.log(err); // Tratamento de mensagens de erro
        }
    }
);

scan.doctor(cb) Verifique o ambiente:

scan.doctor(err => {
    if (err) {
        console.log(err);
    }
});

scan.clone({ newModelName, oldModelName, opts }) Faça uma cópia Model，Pode usar opts O objeto cobre algumas propriedades: 

scan.clone({
    newModelName: 'another-model',
    oldModelName: 'wx-default',
    opts: {}
});

scan.loadModel({ model, udid, port, opts }) Personalize o modelo de entrada: 

const myModel = require('./myModel.js');
scan.loadModel({
    model: myModel,
    udid: '3HX0217705004280',
    port: '4723',
    opts: {
        user: 'xxx',
        pass: 'xxx'
    }
});
